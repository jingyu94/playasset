# PlayAsset 서비스 재기획안 (2026)

## 1. 재설계 원칙
- 기존 구현은 참고만 하고, 원래 의도만 유지한다.
  - 개인 자산 추적
  - 종목/뉴스 기반 의사결정 지원
  - 빠르고 단순한 사용자 흐름
- 폐기 대상
  - 데모성 임시 API 중심 구조
  - `TMP_NEWS_TITLE` 임시 테이블 의존
  - 경계 없는 혼합 라우팅

## 2. 제품 정의
- 제품명: `PlayAsset`
- 제품 성격: 개인 투자자 운영 콘솔
- 핵심 가치
  - 보유 자산/관심종목을 한 화면에서 관리
  - 시장/뉴스 변화가 내 포트폴리오에 미치는 영향 제공
  - 행동이 필요한 시점을 알림으로 전달

## 3. 타깃 사용자와 핵심 과업
- 입문 투자자: "오늘 내 자산에서 바뀐 것만 빠르게 보고 싶다."
- 적극적 개인 투자자: "기회를 빠르게 걸러내고 리스크를 관리하고 싶다."
- 장기 투자자: "투자 가설이 유지되는지 근거 기반으로 확인하고 싶다."

## 4. 핵심 지표
- North-star: `주간 포트폴리오+인사이트 확인 활성 사용자`
- KPI
  - D1/D7 리텐션
  - 알림 클릭률
  - 관심종목 -> 실거래 전환율
  - 인사이트 API p95 지연시간

## 5. 기능 범위
### Release A (필수)
- 계정/인증
- 관심종목 CRUD
- 포트폴리오/계좌/거래내역
- 시세 수집
- 뉴스 수집 + 종목 매핑
- 포트폴리오 대시보드 API

### Release B (필수)
- 종목 단위 감성 요약
- 급등락/거래량/뉴스 급증 이상 감지
- 규칙 기반 알림(가격/키워드/감성)
- 인앱/이메일 알림 전달

### Release C (확장)
- 투자 성향별 전략 템플릿
- 포트폴리오 건강도 점수
- 시그널 리플레이(라이트 백테스트)
- AI 요약 계층(옵션, 독립 서비스)

## 6. 서비스 관점 해석
- 현재 코드는 기술 프로토타입 상태다.
- 목표 서비스는 의사결정 워크플로우 제품이다.
  - 입력: 보유자산, 관심종목, 시세/뉴스 스트림
  - 처리: 정규화, 점수화, 시그널 생성
  - 출력: 우선순위 인사이트, 알림, 근거 설명

## 7. 도메인 경계
- Identity: 사용자/인증/권한/환경설정
- Portfolio: 계좌/포지션/거래/PnL
- Market: 종목마스터/가격/장 상태
- News: 소스/기사/종목언급/감성
- Insight: 스코어링/규칙/알림이벤트
- Notification: 채널별 발송/재시도/결과로그

## 8. 비기능 목표
- 가용성: 99.9%+
- 응답 지연
  - 일반 조회 p95 < 300ms
  - 인사이트 계산 p95 < 1.5s
- 보안
  - 평문 비밀번호 금지
  - 최소권한 DB 접근
  - 인증/중요 쓰기 감사로그
- 운영성
  - 구조화 로그
  - 추적/메트릭
  - 멱등 수집 작업

## 9. 초기 범위 제외
- 증권사 직접 주문 연동
- 레버리지 자동 운용
- 완전자율 AI 트레이딩
