# 구현 실행계획 및 완료내역

## 1단계: 백엔드 런타임 고도화
- 목표
  - Java 21 + Spring Boot 3.x + MVC + VT + Reactive 혼합
  - 고처리량 설정(스레드/커넥션풀/압축/모니터링)
- 완료
  - `Server/playasset/build.gradle` 업그레이드
  - `application.properties` 성능/관측성 설정 추가
  - Actuator + Prometheus 메트릭 노출

## 2단계: 데이터베이스 체계화
- 목표
  - 재설계 스키마 자동 배포
  - 샘플 데이터 자동 주입
- 완료
  - Flyway 마이그레이션 도입
  - `V1__core_schema.sql` 20개 테이블 생성
  - `V2__seed_sample_data.sql` 샘플 데이터 삽입

## 3단계: 서비스 API 확장
- 목표
  - 실서비스형 API(`api/v1`) 제공
  - 대시보드/워치리스트/포지션/알림/거래 등록
- 완료
  - `PlatformController`, `PlatformService`, `PlatformQueryRepository` 구현
  - 검증/예외 처리(`GlobalApiExceptionHandler`) 추가
  - 트랜잭션 등록 시 포지션 자동 반영 로직 적용

## 4단계: 배치/운영 자동화
- 목표
  - 외부 API 키 공란 상태에서도 서비스 시뮬레이션 유지
  - 주기적 시세/뉴스 배치
- 완료
  - `MarketNewsBatchService` 구현
  - 시세 캔들 일괄 upsert 배치
  - 뉴스/감성 샘플 생성 배치
  - `ingestion_jobs` 이력 저장

## 5단계: Flutter 앱 신규 구축
- 목표
  - React Native 대신 Flutter로 프런트 전환
  - 팬시 UI + 실제 API 연동
- 완료
  - `UI/playasset_flutter` 신규 생성
  - Riverpod + Dio + fl_chart 기반 구조
  - 대시보드/관심종목/알림/설정 탭 구현
  - 외부 API 키 공란 정책 문서화

## 6단계: 실행 검증
- 완료
  - Docker 재빌드/재기동 성공
  - Flyway 스키마/시드 적용 성공
  - `api/v1` 엔드포인트 응답 확인
  - 배치 작업 성공 이력 확인
