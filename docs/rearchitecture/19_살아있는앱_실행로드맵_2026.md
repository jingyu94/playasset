# 살아있는 앱 실행 로드맵 (2026-02-16)

## 1. 문서 목적
- 본 문서는 PlayAsset을 "데모 앱"에서 "운영형 살아있는 앱"으로 전환하기 위한 기준 문서입니다.
- 앞으로의 구현/배포/운영 작업은 본 문서의 단계, 우선순위, 완료기준(DoD/KPI)을 따릅니다.

## 2. 최종 목표(북극성)
- 사용자 체감:
  - 앱을 열면 데이터가 "살아 움직인다"는 느낌이 있어야 함
  - 5초 이내 핵심 지표(총자산/당일손익/핵심 액션)가 이해되어야 함
- 기술 체감:
  - UI는 OTA로 무중단 반영
  - 백엔드는 장애 시 핵심 기능이 유지되는 강건한 구조
- 운영 체감:
  - 관리자 화면에서 기준정보/호출정책/권한정책을 제어 가능

## 3. 운영 원칙
- 한국어 문구는 "~요" 톤을 기본으로 유지
- 하드코딩 최소화: 변경 가능성이 있는 문구/룰/임계치는 기준정보(TM_*)로 관리
- 데이터 우선: 인사이트는 근거 수치(샤프/MDD/변동성/집중도)를 반드시 함께 노출
- 모바일 우선: 390x844 기준 가독성, 360폭 이하 대응을 기본 검증으로 사용
- 배포 원칙: Shorebird patch 우선, 네이티브 변경 시에만 APK/AAB 재배포

## 4. 단계별 실행 계획

## 4.1 Phase A (UI/UX 품질 고도화, P0)
- 목표:
  - 화면 밀도 정상화, 정보 계층 재정렬, 금융앱 수준 가독성 확보
- 작업:
  - 대시보드 Hero/요약카드/리스크카드/실행카드 시각 밀도 통일
  - "총자산 + 인라인 수익률 배지(+x.x%)" 패턴 앱 전반 확장
  - 카드 과중첩 제거(큰 박스 + 내부 목록 구조로 정리)
  - 다크/화이트 모드 색 대비 규칙 재정의
  - 도형/글로우 배경 볼륨 축소(시선 방해 최소화)
- 완료기준(DoD):
  - 390x844에서 헤더 과도 줄바꿈/텍스트 겹침 없음
  - 360폭에서 overflow 없음
  - 주요 지표 카드의 1차/2차 정보 구분이 명확함

## 4.2 Phase B (실시간성/개인화 고도화, P0)
- 목표:
  - "정적 샘플" 느낌을 줄이고 사용자 맞춤형 흐름 강화
- 작업:
  - 실시간 갱신 주기 표준화(대시보드, 관심종목, 알림)
  - 투자성향(설문) -> 리밸런싱/ETF 추천 파이프라인 완전 연동
  - 리밸/ETF 반영 변화 미리보기의 종목 구성(매수/매도 금액) 가시성 강화
  - 추천 이유 문구에 정상범위/해석/행동 제안 구조 적용
- 완료기준(DoD):
  - 투자성향 변경 후 추천 결과가 즉시 달라짐
  - 미리보기 카드에서 종목/금액/변화율을 동시에 확인 가능
  - 추천 문구 100% "~요" 톤 + 근거수치 포함

## 4.3 Phase C (운영/관리 기능 완성, P1)
- 목표:
  - 운영자가 코드 수정 없이 정책을 바꿀 수 있어야 함
- 작업:
  - 기준정보 관리 화면(TM_STD_RUNTIME_CONFIG_MAIN) 편집 UX 완성
  - 사용자/권한그룹/권한정책 관리 흐름 완성(대량 사용자 전제)
  - 유료 호출 정책(일 최대 호출 수) 관리 + 사용량 지표 강화
  - 알림 레벨/수신정책 관리 항목 확장
- 완료기준(DoD):
  - 관리자 화면에서 정책 변경 후 앱 동작 반영 확인
  - 호출 한도 초과/차단/재개 시나리오 재현 가능

## 4.4 Phase D (신뢰성/관측성/비용 최적화, P1)
- 목표:
  - 장애에 강하고 운영비 예측 가능한 구조 확보
- 작업:
  - Redis 캐시 계층 표준화(대상/TTL/무효화 정책)
  - LLM/API 호출 캐시, 실패 fallback, 서킷브레이커 적용
  - 핵심 메트릭 대시보드(응답시간/에러율/호출량/캐시적중률) 정의
  - 배치 실패시 재시도/알림 정책 정교화
- 완료기준(DoD):
  - 장애 상황에서 핵심 화면 degrade 모드로 동작
  - 캐시 적중률/호출량 절감 지표를 관측 가능

## 5. 백로그 (우선순위/소유/완료기준)
| 우선순위 | 항목 | 상세 | 완료기준 |
|---|---|---|---|
| P0 | 대시보드 리디자인 2차 | Hero/KPI/요약카드 시각 밸런스 미세튜닝 | 사용자 피드백 기준 "가독성 개선" 확인 |
| P0 | 수익률 배지 공통화 | +/0/- 배지 컴포넌트 공통 사용 | 대시보드/보유자산/추천카드에 동일 규칙 적용 |
| P0 | 화이트 모드 대비 보정 | 저대비 텍스트 전수 점검 | 주요 탭 수동 검수 시 저대비 0건 |
| P0 | 성향-추천 완전연동 | 성향값이 리밸/ETF 추천 가중치에 반영 | 성향 변경 즉시 추천 결과 변화 |
| P1 | 기준정보 관리자 UX | 룰/문구/임계치 편집 워크플로 개선 | 수정/저장/반영까지 1분 이내 가능 |
| P1 | 운영 메트릭 가시화 | 호출량/에러율/응답시간/캐시적중률 | 운영자 관점 대시보드 확인 가능 |
| P1 | 알림정책 확장 | 이벤트 레벨/채널별 수신설정 고도화 | 사용자 맞춤 알림 정책 저장/적용 |
| P2 | 실시간 스트림 고도화 | SSE/WebSocket 검토 및 단계적 도입 | 주요 카드 업데이트 지연 감소 확인 |

## 6. KPI (운영 목표치)
- UX KPI
  - 대시보드 최초 인지 시간: 5초 이내
  - UI 레이아웃 이슈(텍스트 겹침/overflow): 0건
- 기술 KPI
  - 핵심 API p95 응답시간: 500ms 이하
  - 앱 크래시율: 0.5% 이하
  - 캐시 적중률: 60% 이상(LLM/API 대상)
- 운영 KPI
  - 기준정보 변경 후 반영 리드타임: 5분 이내
  - OTA 반영 성공률: 95% 이상

## 7. 테스트/검증 시나리오
- 기능:
  - 로그인(admin/admin), 세션 유지, 재진입
  - 대시보드/관심종목/알림/설정/관리자 탭 정상 렌더링
- UI:
  - 390x844, 360x800, 웹(3000)에서 레이아웃 검증
  - 다크/화이트 전환 시 색 대비 검증
- 데이터:
  - 성향 설문 변경 -> 추천 결과 변화 검증
  - 리밸/ETF 미리보기에서 종목/금액/지표 동시 검증
- 배포:
  - Shorebird patch 배포 후 재설치 없는 반영 확인
  - 서버 로그에서 로그인/대시보드 호출 확인

## 8. 릴리즈/배포 규칙
- 버전 표기:
  - 사용자 커뮤니케이션: `0.0.x` 형태
  - 내부 빌드번호: 자동 관리(`+N`)
- OTA 규칙:
  - patch 시 `API_BASE_URL`은 define-file로 반드시 주입
  - patch는 현재 release-version 대상으로 실행
- 롤백 규칙:
  - 문제 발생 시 즉시 후속 patch 롤포워드
  - 네이티브 이슈인 경우 APK/AAB 재배포 전환

## 9. 수동개입 필요 항목
- 외부 API 실키 연동:
  - 실계정 API 키 발급/등록
  - 호출 과금 정책 확정
- 배포 인프라:
  - 운영 환경 시크릿(API_BASE_URL, 토큰 등) 최종 값 반영
  - 모니터링 알림 채널(Slack/Email/Webhook) 연결
