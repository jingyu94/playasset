# 모바일 앱 스택 결정 가이드 (Flutter vs React Native)

## 결론 요약
- 신규 서비스 기준 기본 추천: `Flutter`
- 단, 팀이 JS/TS 역량이 강하고 RN 레거시를 빠르게 살려야 하면: `React Native`

## Flutter가 유리한 경우
- 앱 UI 일관성과 애니메이션 완성도가 핵심일 때
- 모바일 외 웹/데스크톱까지 단일 코드베이스 확장을 고려할 때
- 네이티브 렌더 편차를 줄이고 싶을 때

## React Native가 유리한 경우
- 기존 RN 코드/라이브러리 자산이 이미 많을 때
- 프런트 팀이 React 생태계 중심일 때
- 네이티브 모듈을 붙이는 작업을 팀이 이미 경험했을 때

## PlayAsset 권장안
- 현재 프로젝트 재설계 시점 기준:
  1. 앱 UX 중심 MVP를 빠르게 완성하려면 Flutter 신규 구축이 유리
  2. 기존 RN 자산 재사용이 목표면 RN 유지 + 최신 아키텍처로 업그레이드
- 실무 선택 기준
  - 6개월 내 출시 속도: 기존 팀 스택 우선
  - 1~2년 유지보수 비용: 인력 수급 가능한 생태계 우선
  - 고급 인터랙션/차트 UX 중요도: Flutter 가중치 상향
